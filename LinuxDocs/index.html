Symbolink vs Hardlink

  • if you delete the original file, a symbolink will be broken, whereas a hard link will remains functional as it points to the same inode
  • Symblink can span file filesystems; while hard links can not
  • Symblink have their own inode and data block, while hard linkks share the the same inode and data blocks as the original file
  • Symbolinks can refernce directories/files while hard link can only reference files
  • In summary symlinks are more flexible and can point directories or across filesystems, but they are more fragile. Hardlinks are more robust
